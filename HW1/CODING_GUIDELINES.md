## LaTeX Rules

### Packages — use only these (safe on any standard installation)
```latex
\usepackage[margin=2.5cm]{geometry}
\usepackage{amsmath, amssymb}
\usepackage{graphicx}
\usepackage{subcaption}
\usepackage{float}
\usepackage{hyperref}
\usepackage{enumitem}
\usepackage{listings}
\usepackage{xcolor}
\usepackage{array}
\usepackage{fancyhdr}
\usepackage[T1]{fontenc}
\usepackage[utf8]{inputenc}
```

### Packages to NEVER use (cause undefined control sequence errors)
- `siunitx` — not available on this installation. Use `\newcommand{\unit}[1]{\,\mathrm{#1}}` instead.
- `booktabs` — not available. Use `\hline` for table rules.
- `titlesec` — causes heading formatting conflicts.
- `parskip` — can interfere with list environments.

### Units — use the custom macro, never \si or \SI
```latex
% Define once in preamble:
\newcommand{\unit}[1]{\,\mathrm{#1}}

% Usage in math mode:
375\unit{mm^2}
42.41\unit{mH}
2.5\unit{A}
106\unit{mN\cdot m}
```

### Degrees — always use ^\circ, never paste the ° Unicode symbol
```latex
% CORRECT:
$\theta = 0^{\circ}$
$90^{\circ}$

% WRONG (causes "Command \textdegree invalid in math mode"):
$\theta = 0°$
```

### Tables — use \hline, never booktabs commands
```latex
% CORRECT:
\begin{tabular}{|l|c|r|}
  \hline
  A & B & C \\
  \hline
  1 & 2 & 3 \\
  \hline
\end{tabular}

% WRONG (booktabs not installed):
\toprule
\midrule
\bottomrule
```

### Table column types — never use S (requires siunitx)
```latex
% CORRECT: use l, c, r
\begin{tabular}{|l|c|r|r|}

% WRONG:
\begin{tabular}{l S S S}
```

### headheight — always set this to avoid fancyhdr warnings
```latex
\setlength{\headheight}{15pt}
```

### Cross-references — always compile TWICE
- First pass builds the aux file.
- Second pass resolves all \ref, \label, \eqref warnings.
- "Undefined references" warnings on first compile are normal.

## MATLAB Rules

### Never use Unicode characters in strings
MATLAB source files are ASCII. Pasting Unicode characters
(·, −, °, →, etc.) causes UTF-8 byte errors, especially when
LaTeX includes the file via \lstinputlisting.

```matlab
% WRONG — causes "Invalid UTF-8 byte" in LaTeX listing:
ylabel('T(\theta)  (mN·m)')
sprintf('−%.1f mN·m', val)

% CORRECT — use ASCII only:
ylabel('T(\theta)  (mN*m)')
sprintf('-%.1f mN*m', val)
```

### Degree symbol in MATLAB axis labels
MATLAB's TeX interpreter supports `^{\circ}` but do not paste `°`:
```matlab
xlabel('Rotor angle \theta (deg)')   % just write "deg"
% or:
xlabel('Rotor angle \theta (^{\circ})')
```

### Plot style conventions for this project
- Line color: blue `'b-'` for inductance, red `'r-'` for torque.
- Peak markers: black filled circles `'ko', 'MarkerFaceColor','k'`.
- No legend — annotate peaks with `text()` directly on the plot.
- No parameter boxes in the figure title or annotation.
- Export: `exportgraphics(fig, 'name.png', 'Resolution', 300)`.

### Figure saving — always use exportgraphics for LaTeX quality
```matlab
exportgraphics(fig1, 'analytical_LT_plots.png', 'Resolution', 300);
```
